### 📚 저희는 이렇게 프로젝트 해요

<hr/>

1. GIT Flow로 효율적으로 관리해요.

브랜치 종류
- main : 운영 서버용 브런치
- release: 개발 서버용, 테스트용 브런치
- develop : 코드 병합 및 테스트용 브런치
- feature : 새로운 기능 개발 브런치
<br> 

- Feature 브랜치 네이밍 규칙
    - {브랜치 종류}/{이름}/{{기능이름}}
    - ex) feature/cahnghyeon/login

2. 노션에 작업상황을 공유하면서 팀원들과 협업하고 체계적으로 프로젝트 해요.
